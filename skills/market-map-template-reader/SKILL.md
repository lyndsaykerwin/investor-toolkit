# Market Map Template Reader

A manifest at `templates/market_map_template.manifest.json` describes the canonical `templates/Market_Map_Template.xlsx` — its sheet, sections, cell addresses, columns, column widths, and styling. Both the template and the manifest are **generated by the renderer** (`render_market_map.py --make-template`), which is the single source of truth for layout — the map has three company tables (Firm Fit Prospects, Market Leaders, Funded Companies) and no Theme Overview block (the theme narrative lives in the Market Brief).

## Rule

When you need the template's structure, read the manifest. Do not inspect the template with openpyxl to discover its layout, cells, fonts, fills, borders, or widths. The manifest already has all of it. Live inspection is redundant work, and its output is re-read on every later turn of the session — so a few inspection calls cost far more than the calls themselves.

## Verify the fingerprint first

```bash
shasum -a 256 "templates/Market_Map_Template.xlsx"
```

Compare the output hash to `template_sha256` in the manifest.

- **Match** → trust the manifest; do not inspect the .xlsx.
- **Mismatch** → the template changed; the manifest is stale. Regenerate both from the renderer: `python skills/render-market-map/scripts/render_market_map.py --make-template` (don't hand-edit the manifest).

That's the whole rule: manifest over inspection, fingerprint to stay honest.

## What this skill does NOT cover

- **Theme-specific populated copies** (e.g., `themes/bi-semantic-layer/BI_Semantic_Layer_Market_Map_vX.xlsx`) — those add columns, rows, merges, and styling on top of the canonical template. The manifest only describes the unpopulated canonical version. If you need the structure of a populated copy, inspect that file directly.
- **What CONTENT goes in each cell** — the manifest is layout-only. Content decisions belong to whatever task you're running (populate, refresh, etc.).
- **Reading or writing the .xlsx for population** — once you know the layout from the manifest, you still use openpyxl to actually write data into cells. The skill saves you the *discovery* step, not the *editing* step.
