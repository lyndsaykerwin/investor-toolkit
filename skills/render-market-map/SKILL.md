# Render market map

The JSONL is the working database; this is the deterministic render step that turns it into the
Excel a human reads. Run the bundled script:

```bash
python skills/render-market-map/scripts/render_market_map.py \
  --in themes/<slug>/<Theme>_Market_Map_vN.jsonl \
  --out themes/<slug>/<Theme>_Market_Map_vN.xlsx \
  --firm "<Firm Name>"
```

Needs `openpyxl` (`pip install openpyxl`). Confirm the script with `--selftest` first.

**What it does.** Places each company into one of three tables (the theme narrative lives in the
Market Brief and the JSONL `_theme_meta` record, not the xlsx):
- **Firm Fit Prospects** — targets that pass your filters (the default bucket).
- **Market Leaders** — public tickers and category-defining incumbents.
- **Funded Companies** — venture-backed scale-ups.

A record can set its own table with `"map_section": "prospects" | "leaders" | "funded"`; otherwise
the script infers it from ownership and funding signals. Output follows banker-grade house style
(navy headers, blue input font, bordered cells, landscape print, no merged cells) and sizes to any
number of companies.

**Rendering invariants** (the script enforces these; keep them true on any edit, verify with `--selftest`):
- Keep every cell legible — title rows never wrap; body columns wrap and each data row sizes to its tallest cell, so text shows in full in Excel, Numbers, and Quick Look alike.
- Show a confirmed `total_funding_usd` of `0` as **Bootstrapped** (an unknown/`null` figure stays blank).
- Show the exact `fte` for headcount; when the source gives only a band, show that band (via `headcount_display`).

Re-run it any time the JSONL changes — the Excel is always a fresh view of the data.

**Canonical template + manifest.** `templates/Market_Map_Template.xlsx` and its `…manifest.json` are
generated by this same renderer — regenerate both after any layout change so they never drift:

```bash
python skills/render-market-map/scripts/render_market_map.py --make-template
```
